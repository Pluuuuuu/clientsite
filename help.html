Basic Text, Password & Hidden Core

Default input type is text. Passwords hide what you type. Hidden fields are not visible but are submitted with the form.

type="text" type="password" type="hidden"
What you see
Text fields show input, password masks it, hidden is not shown.
Hidden field: sent with form but not visible here
The code
<!-- default type is text -->
<label for="fname">First name:</label>
<input type="text" id="fname" name="fname" value="John">

<label for="lname">Last name:</label>
<input type="text" id="lname" name="lname" value="Doe">

<label for="pwd">Password:</label>
<input type="password" id="pwd" name="pwd">

<!-- not visible in the page, but submitted -->
<input type="hidden" id="custId" name="custId" value="3487">
  • If you omit type, it defaults to text.
  • password just masks characters; it is not "security" alone.
  • hidden can be seen/edited from DevTools, so not for secrets.

Submit, Reset, Button & Image Button Actions

These inputs trigger actions: submitting a form, resetting its values, or running JavaScript.

type="submit" type="reset" type="button" type="image"
What you see
Try clicking the alert button. Image button will submit a form like a normal submit (no real img file here).
The code
<input type="submit" value="Submit Form">
<input type="reset"  value="Reset Form">

<input
  type="button"
  value="Click Me (Alert)"
  onclick="alert('Hello World!')"
/>

<input
  type="image"
  src="img_submit.gif"
  alt="Submit via Image"
  width="48"
  height="48"
/>
  • submit: sends the form to the URL in action.
  • reset: restores default values of the form fields.
  • button: does nothing by itself – needs JS (onclick).
  • image: submit button that uses an image.

Radio Buttons vs Checkboxes Choices

Radios allow only one choice within the same name. Checkboxes allow any combination.

type="radio" type="checkbox"
What you see
Radio group: pick one language. Checkboxes: pick zero, one, or many vehicles.

Favorite web language (radio):

Vehicles (checkbox):

The code
<!-- radio: same name => only one selected -->
<input type="radio" id="html" name="fav_language" value="HTML">
<label for="html">HTML</label>

<input type="radio" id="css" name="fav_language" value="CSS">
<label for="css">CSS</label>

<input type="radio" id="javascript" name="fav_language" value="JavaScript">
<label for="javascript">JavaScript</label>

<!-- checkbox: independent boolean flags (different names) -->
<input type="checkbox" id="vehicle1" name="vehicle1" value="Bike">
<label for="vehicle1">I have a bike</label>

<input type="checkbox" id="vehicle2" name="vehicle2" value="Car">
<label for="vehicle2">I have a car</label>

<input type="checkbox" id="vehicle3" name="vehicle3" value="Boat">
<label for="vehicle3">I have a boat</label>
  • Same name + radio = only one can be selected.
  • Checkboxes are separate booleans: each one can be checked/unchecked independently.

Color Picker UI

Opens a native color picker. The value is usually in hex (#rrggbb).

type="color"
What you see
Click to choose a color. Not all older browsers support this.
The code
<label for="favcolor">Select your favorite color:</label>
<input type="color" id="favcolor" name="favcolor">
  • Value looks like #ff0000 (red), etc.

Date & Time Inputs HTML5

HTML5 adds several date/time-related types. They often show native pickers and support min/max limits.

type="date" type="datetime-local" type="month" type="time" type="week"
What you see
Date and time pickers with min/max where applicable.
The code
<input type="date" id="birthday" name="birthday">

<input
  type="date"
  id="datemax"
  name="datemax"
  max="1979-12-31"
/>

<input
  type="date"
  id="datemin"
  name="datemin"
  min="2000-01-02"
/>

<input
  type="datetime-local"
  id="birthdaytime"
  name="birthdaytime"
/>

<input type="month" id="bdaymonth" name="bdaymonth">
<input type="time"  id="appt"      name="appt">
<input type="week"  id="week"      name="week">
  • min / max restrict allowed values.
  • datetime-local has date + time but no timezone.
  • week selects "week number + year".

Email, Telephone, URL & Search Semantic

These types give the browser semantic hints and automatic validation (for email & URL).

type="email" type="tel" type="url" type="search"
What you see
Email / URL are validated on submit. tel is usually used with a pattern.
The code
<label for="email">Enter your email:</label>
<input type="email" id="email" name="email">

<label for="phone">Enter your phone (000-00-000):</label>
<input
  type="tel"
  id="phone"
  name="phone"
  pattern="[0-9]{3}-[0-9]{2}-[0-9]{3}"
  placeholder="123-45-678"
/>

<label for="homepage">Homepage (URL):</label>
<input type="url" id="homepage" name="homepage">

<label for="gsearch">Search:</label>
<input type="search" id="gsearch" name="gsearch">
  • email and url have built-in browser validation.
  • pattern is used to manually validate tel formats.
  • search behaves like text but may show a "clear" icon.

File Upload Files

Allows the user to choose files to upload. Needs form enctype="multipart/form-data" on real servers.

type="file"
What you see
Browse your filesystem and pick a file.
The code
<label for="myfile">Select a file:</label>
<input type="file" id="myfile" name="myfile">
  • To allow multiple files: add multiple.

Number & Range (Slider) Numeric

Number has up/down arrows and enforces numeric input. Range is a slider; the exact value is less important in the UI.

type="number" type="range" min / max / step
What you see
Use min, max, and step to control allowed values.
The code
<input
  type="number"
  id="quantity1"
  name="quantity1"
  min="1"
  max="5"
/>

<input
  type="number"
  id="quantity2"
  name="quantity2"
  min="0"
  max="100"
  step="10"
  value="30"
/>

<input
  type="range"
  id="vol"
  name="vol"
  min="0"
  max="50"
/>
  • step controls the increment for number/range.
  • Min/max are a kind of validation constraints.

Common Input Attributes Validation

Attributes like required, readonly, disabled, maxlength, size, pattern and checked change behaviour and validation.

required readonly disabled maxlength size pattern checked
What you see
Try to submit without filling the required field (in a real form).
The code
<input
  type="text"
  id="requiredText"
  name="requiredText"
  required
  placeholder="You must fill this"
/>

<input
  type="text"
  id="readonlyText"
  name="readonlyText"
  value="You cannot edit me"
  readonly
/>

<input
  type="text"
  id="disabledText"
  name="disabledText"
  value="I am disabled"
  disabled
/>

<input
  type="text"
  id="shortText"
  name="shortText"
  maxlength="10"
  placeholder="Max 10 chars"
/>

<input
  type="text"
  id="wideText"
  name="wideText"
  size="30"
  placeholder="Visually wider field"
/>

<input
  type="text"
  id="patternText"
  name="patternText"
  pattern="[A-Za-z]{3}[0-9]{3}"
  placeholder="e.g. abc123"
/>

<input
  type="checkbox"
  id="prechecked"
  name="prechecked"
  value="yes"
  checked
/>
  • required: browser blocks submit if empty.
  • readonly: user cannot change it, but it is submitted.
  • disabled: user cannot interact and it is not submitted.
  • maxlength: limits characters; size is just visual width.
  • pattern: regex the value must match.
  • checked: default state for checkbox/radio.
Remember for exams: know what each type does, common attributes, and how validation works in the browser.
